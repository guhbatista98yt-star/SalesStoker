## 1. Filosofia e Princípios

### Mobile-First com Desktop Premium

| Princípio | Regra prática |
|---|---|
| **Mobile-first** | Estilos base para mobile, sobrescreva com `sm:`, `md:`, `lg:` |
| **Touch-friendly** | Nenhum alvo de toque menor que 44×44px em mobile |
| **Progressive disclosure** | Mostre o essencial em mobile; expanda detalhes no desktop |
| **Zero scroll horizontal** | Jamais. Tabelas ocultam colunas, nunca adicionam `overflow-x-auto` |
| **Safe areas** | Sempre respeitar `env(safe-area-inset-*)` em mobile |
| **100dvh** | Usar `h-[100dvh]` (não `100vh`) — evita barra de endereço em mobile |
| **Consistência de tokens** | Mesmos tokens em todos os breakpoints — apenas layout muda |

### Dois modos de uso

```
MODO OPERACIONAL (mobile)          MODO GESTÃO (desktop)
─────────────────────────          ──────────────────────
Full-screen h-[100dvh]             Shell com sidebar
Sem sidebar                        AppBar (70px) + sidebar fixa
CTAs: w-full h-14 rounded-xl       Botões: h-9 px-4
Fonte: text-sm/text-base           Fonte: text-sm
1 coluna                           2–4 colunas
Scan / câmera / barcode            Mouse + teclado + search
```

## 2. Breakpoints e Grid

### Sistema de breakpoints

| Prefixo | Min-width | Comportamento |
|---|---|---|
| *(base)* | 0px | Mobile — 1 coluna, sem sidebar |
| `sm:` | 640px | Mobile landscape — algumas colunas começam |
| `md:` | 768px | Tablet — nome do usuário aparece no chip |
| `lg:` | 1024px | Desktop — sidebar aparece, drawer some |
| `xl:` | 1280px | Desktop wide — colunas extras nas tabelas |
| `2xl:` | 1536px | Desktop ultra-wide |

### Grids responsivas prontas

```tsx
// Tiles de módulos (ActionTile)
<div className="grid grid-cols-3 sm:grid-cols-4 md:grid-cols-5 lg:grid-cols-6 gap-2 md:gap-3">

// Stats strip (KPIs)
<div className="grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 gap-3 md:gap-4">

// Formulário
<div className="grid grid-cols-1 sm:grid-cols-2 gap-4">

// Lista + painel de filtros lateral
<div className="grid grid-cols-1 lg:grid-cols-[280px_1fr] gap-6">

// Detalhes / cards info
<div className="grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-4">
```

## 4. Temas de Cor — ThemeCustomizer

O sistema tem **5 temas** aplicados via `data-logo-theme` no `<html>`. Mudam a cor primária, sidebar e todos os componentes que usam `--primary`, `--secondary`, `--ring` etc.

### Como funciona

1. `applyLogoTheme(id)` → `document.documentElement.setAttribute("data-logo-theme", id)`
2. `localStorage.setItem("wms-logo-theme", id)` → persiste entre sessões
3. CSS aplica os overrides de tokens por `html[data-logo-theme="..."]`

### O que muda por tema

Quando o tema é trocado, **todos** esses elementos mudam de cor automaticamente por usarem CSS custom properties:

| Elemento | Token usado |
|---|---|
| Botão primário | `--primary` / `--primary-foreground` |
| Item selecionado no menu | `--secondary` / `--secondary-foreground` |
| Hover do menu | `--secondary/60` |
| Badge ativo, ring de focus | `--ring` |
| Avatar do usuário (fundo) | `--primary/20` |
| Gear button | `bg-primary` |
| Chip do usuário (iniciais) | `text-primary` |
| Indicador ativo nos charts | `--chart-1` |
| Botão de toggle (claro/escuro ativo) | `border-primary bg-primary/10 text-primary` |
| Borda do input em focus | `focus:ring-primary/20 focus:border-primary/40` |
| `card-hover:hover` borda | `hsl(var(--primary) / 0.25)` |
| AppBar icon button glow no hover | `hsl(var(--primary) / 0.12)` |

## 6. Espaçamento e Ritmo

### Padding por contexto

| Contexto | Classes |
|---|---|
| Wrapper de página (gestão) | `px-4 py-4 md:px-6 md:py-6` |
| Wrapper de página (operacional mobile) | `px-4 py-4` com `max-w-lg mx-auto` |
| Card interno | `p-4` (mobile) / `p-6` (desktop) |
| Item de lista compacto | `px-4 py-3` |
| AppBar / header fixo | `h-[70px]` |
| Gap entre cards | `gap-3 md:gap-4` |
| Gap entre campos de formulário | `gap-3 sm:gap-4` |
| Gap entre tiles de módulos | `gap-2 md:gap-3` |
| Safe area iOS (bottom) | `pb-[max(env(safe-area-inset-bottom),16px)]` |

### Classe utilitária `.safe-bottom`

```css
.safe-bottom {
  padding-bottom: max(env(safe-area-inset-bottom, 0px), 16px);
}
```

```css

@media (min-width: 1024px) {
```

## 8. Efeitos de Interação — hover / press / focus

### `.hover-elevate` e `.active-elevate-2`

Efeito de overlay aplicado automaticamente em botões e badges via pseudo-elemento `::after`.

```css
/* Aplicado por padrão em botões (via shadcn button variants) */
.hover-elevate:hover::after    { background-color: var(--elevate-1); }  /* 3% overlay */
.active-elevate-2:active::after { background-color: var(--elevate-2); } /* 8% overlay */
```

### `.nav-item` — item de menu

```css
.nav-item {
  transition: background-color 150ms ease, color 150ms ease, transform 120ms ease;
}
.nav-item:active { transform: scale(0.97); }
```

**Estado ativo (página atual):**
```tsx
className={cn(
  "nav-item flex items-center gap-[13px] px-3 rounded-lg mb-0.5 cursor-pointer py-[10px]",
  isActive
    ? "bg-secondary text-secondary-foreground font-semibold"
    : "text-foreground hover:bg-secondary hover:text-secondary-foreground"
)}
```

**Estado mini (só ícone, 46×46px):**
```tsx
className={cn(
  "nav-item flex items-center justify-center rounded-lg mb-0.5 cursor-pointer w-[46px] h-[46px]",
  isActive
    ? "bg-secondary text-secondary-foreground"
```

## 11. ThemeCustomizer — Engrenagem Flutuante

O ThemeCustomizer tem 3 partes: **gear button flutuante**, **backdrop**, e **painel deslizante**.

### Gear button

```
Posição: fixed, z-1300
Tamanho: 44×44px (w-11 h-11) — touch-friendly
Shape: rounded-full
Cor: bg-primary text-primary-foreground
Shadow: shadow-md
Cursor: grab / grabbing durante drag
```

**Comportamento drag + tap:**
- **Mouse:** threshold 4px — abaixo disso é clique, abre o painel
- **Touch:** threshold 10px — maior para evitar falsos positivos (tremor do dedo)
- `e.preventDefault()` no touchmove — evita competição com scroll da página
- Posição persiste em `localStorage("wms:gear-pos")`
- Posição padrão: `{ x: window.innerWidth - 52, y: window.innerHeight / 2 - 22 }` (lado direito, meio da tela)

```tsx
<button
  ref={btnRef}
  onMouseDown={handleDragStart}
  onTouchStart={handleDragStart}
  style={{ position: "fixed", left: pos.x, top: pos.y, zIndex: 1300 }}
  className="w-11 h-11 flex items-center justify-center rounded-full bg-primary text-primary-foreground shadow-md select-none touch-none"
>
  <Settings className="gear-spin h-5 w-5" strokeWidth={1.8} />
</button>
```

## 18. Prompt para IA

Copie e cole em qualquer IA (ChatGPT, Claude, Cursor, Lovable, v0, etc.):

---

> **Crie uma interface React + TypeScript + Tailwind CSS totalmente responsiva (mobile e desktop) seguindo este design system:**
>
> ---
>
> **FILOSOFIA**
> - Mobile-first obrigatório. Estilos base = mobile. `sm:` `md:` `lg:` escalam para cima.
> - Dois modos: operacional (full-screen mobile, `h-[100dvh]`, sem sidebar) e gestão (desktop com shell: AppBar 70px + sidebar).
> - Zero scroll horizontal. Tabelas ocultam colunas progressivamente.
> - `h-[100dvh]` em páginas full-screen mobile (não `100vh` — barra do navegador quebra).
> - `safe-bottom` class (`pb-[max(env(safe-area-inset-bottom),16px)]`) em toda página scrollável mobile.
>
> **CORES LIGHT**
> - Fundo: `hsl(220 17% 95%)` azul-cinza #f0f2f5
> - Cards: `hsl(0 0% 100%)` branco
> - Primária: `hsl(260 54% 45%)` roxo #5e35b1
> - Hover/selecionado: `hsl(264 46% 94%)` roxo clarinho #ede7f6
> - Texto: `hsl(220 22% 14%)`
> - Muted: `hsl(220 10% 46%)`
>
> **CORES DARK**
> - Fundo: `hsl(229 37% 14%)` navy #1a2035
> - Cards: `hsl(226 34% 17%)`
> - Primária: `hsl(258 90% 68%)` roxo #7c4dff
>
> **TIPOGRAFIA**
> - Poppins (Google Fonts, pesos 300–800). Nunca Inter ou Geist como principal.
> - Corpo: `text-sm`. Títulos: `font-bold`. Seções: `text-[11px] font-bold uppercase tracking-[0.12em]`.
>
> **BORDER RADIUS** — Cards: `rounded-xl` (12px). Botões: `rounded-xl`. Badges: `rounded-md`.
>
> **SOMBRAS** — Berry style (wide-spread, suavíssimas): `0 2px 14px 0 rgb(32 40 45 / 10%)`. NO dark: `0 2px 14px 0 rgb(0 0 0 / 30%)`.
>
> **BOTÕES**
> - CTA mobile: `w-full h-14 rounded-xl font-semibold shadow-lg shadow-primary/15 active:scale-[0.98] transition-transform`
> - Desktop padrão: `h-9 px-4 rounded-xl`
> - Mínimo mobile: `h-11` (44px)
> - Todas as variantes herdam `hover-elevate active-elevate-2` (overlay via `::after`, não muda cor de fundo)
>
> **INPUTS** — `h-11 rounded-xl` mobile. `h-9 rounded-xl` desktop. Busca: `pl-9` com ícone absoluto.
>
> **NAVEGAÇÃO DESKTOP (Shell)**
> - AppBar fixo, `h-[70px]`, `z-[1100]`, `bg-card border-b`
> - Sidebar fixa esquerda (ou direita), `z-[1099]`, `bg-card border-r`
> - Estados: mini 72px (só ícones 46×46px + Tooltip `side="right"`) ↔ expandida 260px (ícones + labels)
> - Transição de largura: `300ms cubic-bezier(0.4,0,0.2,1)`
> - Item ATIVO: `bg-secondary text-secondary-foreground font-semibold`
> - Item hover: `hover:bg-secondary hover:text-secondary-foreground`
> - NavItem press: `transform scale(0.97) 120ms`
> - Grupos colapsáveis com CSS grid trick: `grid-template-rows: 0fr → 1fr`
> - Chevron do grupo: `transition-transform 220ms` + `rotate-0` / `-rotate-90`
> - Logo sidebar aparece com: `opacity:0 + translateX(-8px) → normal` em 220ms
>
> **NAVEGAÇÃO MOBILE**
> - AppBar com botão ☰ abre Drawer 260px (slide da esquerda `translateX(-100%) → 0` em 240ms)
> - Overlay: `bg-black/50` fade-in 180ms
> - Botão ⊞ na AppBar (ao lado do chip do usuário) abre painel de configurações — `lg:hidden`
> - Drawer fecha ao clicar qualquer item de menu
>
> **CUSTOMIZADOR DE TEMA**
> - Engrenagem flutuante: `w-11 h-11 rounded-full bg-primary text-primary-foreground shadow-md`, `z-[1300]`
> - Posição inicial: canto direito, meio da altura da tela
> - Drag: threshold 4px mouse / 10px touch — abaixo = clique (abre painel)
> - `e.preventDefault()` no touchmove para não competir com scroll
> - Painel: slide da direita, `w-full sm:w-[300px]`, `z-[1500]`, `bg-card border-l`
> - Engrenagem gira: `animation: gear-spin 6s linear infinite`
>
> **TABELAS**
> - `hidden sm:table-cell` (≥640px), `hidden md:table-cell` (≥768px), `hidden lg:table-cell` (≥1024px)
> - ID/nome e botões de ação: SEMPRE visíveis
> - Status badge: SEMPRE visível
> - Container: `rounded-xl border border-border/50 bg-card overflow-hidden`
>
> **GRIDS**
> - Tiles: `grid-cols-3 sm:grid-cols-4 md:grid-cols-5 lg:grid-cols-6`
> - Stats: `grid-cols-2 sm:grid-cols-3 lg:grid-cols-4`
> - Formulário: `grid-cols-1 sm:grid-cols-2`
> - Lista+filtros: `grid-cols-1 lg:grid-cols-[280px_1fr]`
>
> **EFEITOS HOVER/PRESS**
> - Cards com lift: `card-hover` class — `translateY(-1px) + shadow-md + border-color primary/25` em 200ms
> - Tiles: `active:scale-[0.97] active:bg-muted/50 transition-all duration-150`
> - AppBar buttons: `hover: box-shadow primary/12` + `active: scale(0.94)` em 120ms
> - Botões: overlay `::after` de 3% hover / 8% press (não altera background)
>
> **BADGES DE STATUS**
> - `border + text + bg` coloridos + variante dark mode
> - Pattern: `border-X-200 text-X-700 bg-X-50 dark:border-X-800 dark:text-X-300 dark:bg-X-950/40`
> - Cores: yellow=pendente, blue=em_andamento, green=concluido, red=cancelado, purple=finalizado
>
> **ANIMAÇÕES**
> - Página: `opacity:0 + translateY(6px) → normal` em 200ms (classe `animate-page-enter`)
> - Modais: `scale(0.95) + opacity` em 250ms (classe `animate-scale-in`)
> - Cards slide: `opacity:0 + translateY(12px) → normal` em 400ms (classe `animate-slide-up`)
>
> **REGRAS GERAIS**
> - Ícones: APENAS `lucide-react`, `strokeWidth={1.5}` (1.8 em ícones de destaque)
> - Dark mode via classe `.dark` no `<html>` (não via `prefers-color-scheme`)
> - CSS custom properties: formato `H S% L%` com espaços (não `hsl(...)`)
> - Scrollbar: `width: 6px`, thumb `muted-foreground/20`, hover `muted-foreground/35`
