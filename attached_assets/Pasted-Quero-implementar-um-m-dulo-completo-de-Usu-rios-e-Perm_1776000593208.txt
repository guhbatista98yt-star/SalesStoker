Quero implementar um módulo completo de Usuários e Permissões dentro de Configurações, com padrão profissional, totalmente personalizável, escalável e com governança real de acesso.

IMPORTANTE:
Hoje os vendedores acessam usando um código, mas isso está limitado demais.
Quero evoluir a aplicação para um modelo correto de sistema profissional, onde seja possível:
- criar usuários
- definir perfil
- controlar acesso por função
- controlar acesso por módulo
- controlar acesso por ação
- controlar acesso por dados/resultados
- controlar quem pode visualizar, editar, aprovar, exportar ou alterar informações

Ou seja:
quero transformar o acesso da aplicação em um sistema robusto de gestão de usuários, perfis e permissões, mantendo a simplicidade operacional quando necessário.

============================================================
1. OBJETIVO DO MÓDULO
============================================================

Criar em Configurações um módulo chamado algo como:
- Usuários
- Usuários e Permissões
- Controle de Acesso

Esse módulo deve permitir:
- criar usuários
- editar usuários
- ativar/inativar usuários
- bloquear usuários
- redefinir acesso
- criar perfis
- editar perfis
- criar permissões personalizadas
- vincular usuários a perfis
- controlar acesso por módulo, função, resultado e alteração
- auditar alterações de acesso

Quero algo com padrão enterprise, profissional e fácil de manter.

============================================================
2. MANTER COMPATIBILIDADE COM O ACESSO ATUAL
============================================================

Hoje os vendedores acessam por código.
Quero que você avalie a melhor arquitetura para manter isso de forma profissional.

Possibilidades:
- manter o código como identificador/login simplificado para vendedor
- permitir usuário + senha para perfis administrativos
- permitir que o vendedor também tenha usuário vinculado ao seu código
- tratar o código como credencial operacional vinculada a um usuário real no sistema

Quero a solução mais inteligente e profissional, sem perder praticidade.

============================================================

Quero permissões em nível fino.

Separar permissões por:

A. Módulo
- dashboard
- campanhas
- criação de campanhas
- comissão
- metas
- relatórios
- usuários
- configurações
- tela do vendedor
- dashboard do supervisor
- exportações
- qualquer outro módulo existente

B. Ação
- visualizar
- criar
- editar
- excluir
- ativar
- pausar
- encerrar
- aprovar
- exportar
- recalcular
- simular
- alterar regra
- alterar configuração
- ver memória de cálculo
- ver valores financeiros

C. Escopo de dados
- ver apenas os próprios dados
- ver a própria equipe
- ver a própria loja
- ver múltiplas lojas
- ver tudo
- ver apenas campanhas em que participa
- ver apenas resultados consolidados
- ver resultado detalhado
- ver comissão própria
- ver comissão da equipe
- ver comissão geral
- ver relatórios financeiros ou não

Quero que a permissão seja realmente profissional, não apenas “acessa ou não acessa”.

============================================================
6. EXEMPLOS DE REGRAS DE ACESSO
============================================================

Quero conseguir configurar cenários como:

- vendedor vê apenas suas campanhas, seus resultados, sua comissão e sua evolução
- vendedor não pode editar campanhas, metas, comissões ou usuários
- supervisor vê a equipe dele, resultados da equipe, campanhas da equipe e dashboards gerenciais
- supervisor não pode alterar regras estruturais críticas sem permissão
- gerente vê loja/unidade inteira
- financeiro pode ver valores, exportações e comissões
- marketing/fornecedor pode ver apenas campanhas específicas ou resultados autorizados
- administrador pode tudo
- perfil customizado pode combinar acessos específicos
