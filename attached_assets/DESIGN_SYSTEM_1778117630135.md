## 1. Filosofia e Princípios

### Mobile-First com Desktop Premium

O sistema parte do mobile e escala para o desktop — nunca o contrário.

| Princípio | Regra prática |
|---|---|
| **Mobile-first** | Escreva estilos base para mobile, sobrescreva com `sm:`, `md:`, `lg:` |
| **Touch-friendly** | Nenhum alvo de toque menor que 44×44px em mobile |
| **Progressive disclosure** | Mostre o essencial em mobile; expanda detalhes no desktop |
| **Zero scroll horizontal** | Jamais. Tabelas ocultam colunas, nunca adicionam `overflow-x-auto` |
| **Safe areas** | Sempre respeitar `env(safe-area-inset-*)` em mobile |
| **100dvh** | Usar `h-[100dvh]` (não `100vh`) para evitar barra de endereço em mobile |
| **Consistência** | Mesmos tokens em todos os breakpoints — apenas layout muda |

### Dois modos de uso

```
MODO OPERACIONAL (mobile)          MODO GESTÃO (desktop)
─────────────────────────          ──────────────────────
Full-screen                        Shell com sidebar
Sem sidebar                        AppBar + sidebar fixa
CTAs grandes (h-14)                Botões compactos (h-9)
Fonte maior (text-base)            Fonte menor (text-sm)
1 coluna                           2–4 colunas
Scan / câmera / barcode            Mouse + teclado
```

## 2. Breakpoints e Grid

### Sistema de breakpoints (Tailwind padrão)

| Prefixo | Min-width | Dispositivo típico |
|---|---|---|
| *(base)* | 0px | Mobile pequeno (320px+) |
| `sm:` | 640px | Mobile grande / landscape |
| `md:` | 768px | Tablet portrait |
| `lg:` | 1024px | Tablet landscape / laptop |
| `xl:` | 1280px | Desktop |
| `2xl:` | 1536px | Desktop wide |

### Grid responsiva padrão

```tsx
// Dashboard — tiles de módulos
<div className="grid grid-cols-3 sm:grid-cols-4 md:grid-cols-5 lg:grid-cols-6 gap-3">

// Cards de métricas (stats strip)
<div className="grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 gap-4">

// Layout de conteúdo com sidebar de filtros
<div className="grid grid-cols-1 lg:grid-cols-[280px_1fr] gap-6">

// Formulário de cadastro
<div className="grid grid-cols-1 sm:grid-cols-2 gap-4">

// Detalhes / 3 colunas
<div className="grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-4">
```

### Larguras máximas de conteúdo

| Contexto | Classe |
|---|---|
| Página operacional mobile | `max-w-lg mx-auto` |
| Formulário / login | `max-w-sm mx-auto` ou `max-w-md mx-auto` |
| Página de gestão | `max-w-7xl mx-auto` (com padding lateral) |
| Conteúdo editorial | `max-w-3xl mx-auto` |
| Full width | `w-full` (sem max-w) |

## 5. Tipografia

### Família e pesos

```css
/* Google Fonts — carregar no <head> ou @import */
@import url('https://fonts.googleapis.com/css2?family=Poppins:wght@300;400;500;600;700;800&family=Inter:wght@400;500;600&display=swap');
```

### Escala tipográfica responsiva

| Elemento | Mobile | Desktop | Peso | Classe Tailwind |
|---|---|---|---|---|
| Hero / Display | `text-2xl` | `text-4xl` | 800 | `font-extrabold` |
| Título de página | `text-xl` | `text-2xl` | 700 | `font-bold` |
| Título de card | `text-base` | `text-lg` | 600 | `font-semibold` |
| Subtítulo / Label | `text-sm` | `text-sm` | 500 | `font-medium` |
| Corpo principal | `text-sm` | `text-sm` | 400 | `font-normal` |
| Texto auxiliar / hint | `text-xs` | `text-xs` | 400 | `text-muted-foreground` |
| Label de input | `text-sm` | `text-sm` | 500 | `font-medium` |
| Badge / Chip | `text-xs` | `text-xs` | 600 | `font-semibold` |
| Código / barcode | `text-sm` | `text-sm` | 400 | `font-mono` |

### Aplicação

```tsx
// Título de página
<h1 className="text-xl md:text-2xl font-bold text-foreground tracking-tight">
  Título
</h1>

// Subtítulo / breadcrumb
<p className="text-sm text-muted-foreground">
  Descrição ou caminho
</p>

// Número grande em destaque (dashboard)
<span className="text-4xl md:text-5xl font-bold tracking-tighter text-foreground">
  1.240
</span>

// Label de seção
<p className="text-[10px] md:text-xs font-semibold uppercase tracking-wider text-muted-foreground/60">
  Seção
</p>
```

## 6. Espaçamento e Ritmo

### Escala de padding interno por contexto

| Contexto | Mobile | Desktop |
|---|---|---|
| Página (wrapper) | `px-4 py-4` | `px-6 py-6` |
| Card interno | `p-4` | `p-6` |
| Card compacto (item lista) | `px-4 py-3` | `px-4 py-3` |
| Header fixo de página | `px-4 h-14` | `px-6 h-14` |
| AppBar | `px-4 h-[70px]` | `px-4 h-[70px]` |
| Formulário — gap entre campos | `gap-3` | `gap-4` |
| Seção — gap entre cards | `space-y-3` | `space-y-4` |
| Grid de módulos / tiles | `gap-2` | `gap-3` |
| Safe area (iOS) | `pb-[max(env(safe-area-inset-bottom),16px)]` | — |

### Classe utilitária `.safe-bottom`

```css
.safe-bottom {
  padding-bottom: max(env(safe-area-inset-bottom, 0px), 16px);
}
```

### Padrão de página operacional (mobile)

```tsx
<div className="max-w-lg mx-auto px-4 py-4 space-y-3 safe-bottom">
  {/* conteúdo */}
</div>
```

### Padrão de página de gestão (desktop)

```tsx
<div className="px-4 py-4 md:px-6 md:py-6 max-w-7xl mx-auto space-y-4 md:space-y-6">
  {/* conteúdo */}
</div>
```

## 7. Elevação e Sombras

As sombras seguem o padrão Berry: largas, suaves, com blur wide-spread.

| Token | Tailwind aprox. | Uso |
|---|---|---|
| `--shadow-2xs` | `shadow-[0_1px_3px...]` | Inputs, bordas sutis |
| `--shadow-sm` | `shadow-sm` | Cards em repouso |
| `--shadow` | `shadow` | Cards com hover ligeiro |
| `--shadow-md` | `shadow-md` | Cards hover / dropdowns |
| `--shadow-lg` | `shadow-lg` | Modais, drawers |
| `--shadow-xl` | `shadow-xl` | Sheets, popovers grandes |

```css
/* Uso via var() */
box-shadow: var(--shadow);
box-shadow: var(--shadow-md);
```

## 8. Componentes

### 8.1 Botões

```tsx
// Primário — ação principal
<Button>Confirmar</Button>

// Secundário
<Button variant="secondary">Cancelar</Button>

// Destrutivo
<Button variant="destructive">Excluir</Button>

// Contorno
<Button variant="outline">Filtrar</Button>

// Ghost (ícone ou ação terciária)
<Button variant="ghost" size="icon"><X /></Button>

// CTA mobile — botão de ação principal em telas operacionais
<Button className="w-full h-14 rounded-xl text-base font-semibold shadow-lg shadow-primary/15 active:scale-[0.98] transition-transform">
  Confirmar Pedido
</Button>

// Botão compacto desktop
<Button size="sm">Exportar</Button>

// Ícone + label mobile (touch target garantido)
<Button className="h-12 gap-2 px-5">
  <Check className="h-5 w-5" />
  Confirmar
</Button>
```

**Regras:**
- Mobile: mínimo `h-11` (44px) para todos os botões interativos
- Desktop: `h-9` (padrão) é aceitável
- Todos herdam `hover-elevate active-elevate-2` (efeito overlay via `::after`)
- CTA principal em mobile: sempre `w-full h-14 rounded-xl`

> **Crie uma interface React + TypeScript + Tailwind CSS totalmente responsiva (mobile e desktop) com as seguintes diretrizes de design:**

> - Dois modos: operacional (full-screen mobile, sem sidebar) e gestão (desktop com sidebar)
>

> - Background: `hsl(220 17% 95%)` — azul-cinza suave
> - Cards: branco `hsl(0 0% 100%)`
> - Primária: roxo `hsl(260 54% 45%)` (#5e35b1)

> - Texto: `hsl(220 22% 14%)`
> - Muted: `hsl(220 10% 46%)`

> **PALETA — DARK**
> - Background: `hsl(229 37% 14%)` — navy #1a2035
> - Cards: `hsl(226 34% 17%)`
> - Primária: roxo claro `hsl(258 90% 68%)` (#7c4dff)
>
> **TIPOGRAFIA**
> - Fonte: Poppins (Google Fonts, pesos 300–800) — sem Inter ou Geist
> - Corpo: `text-sm font-normal`
> - Títulos: `font-bold` ou `font-semibold`
> - Labels: `text-xs font-medium uppercase tracking-wider text-muted-foreground`
>
> **BORDER RADIUS**
> - Cards: `rounded-xl` (12px)
> - Botões e inputs: `rounded-xl` (também 12px)
> - Badges: `rounded-md` (8px)
>
> **SOMBRAS** — suaves, wide-spread: `0 2px 14px 0 rgb(32 40 45 / 10%)`
>
> **BOTÕES**
> - Primary: `bg-primary text-white border border-primary-border hover-elevate active-elevate-2`
> - Mobile CTA: `w-full h-14 rounded-xl font-semibold shadow-lg shadow-primary/15 active:scale-[0.98] transition-transform`
> - Desktop: `h-9 px-4 rounded-xl`
> - Mínimo mobile: `h-11` (44px)
>
> **INPUTS**
> - Mobile: `h-11 rounded-xl`
> - Desktop: `h-10 rounded-xl`
> - Busca com ícone leading, `pl-9`
>
> **NAVEGAÇÃO DESKTOP**
> - Sidebar fixa à esquerda, 72px (mini, só ícones com tooltip) ↔ 260px (expandida, ícones + labels)

> - AppBar 70px com botão ☰ que abre drawer 260px (slide-left + overlay)
> - Drawer: `animation: translateX(-100%→0) 240ms cubic-bezier(0.4,0,0.2,1)`
> - Overlay: `bg-black/50`
> - Alternativa: bottom navigation bar com 4-5 tabs
>
> **LAYOUT CONTEÚDO**
> - Mobile: `max-w-lg mx-auto px-4 py-4 space-y-3 pb-[max(env(safe-area-inset-bottom),16px)]`
> - Desktop: `px-6 py-6 max-w-7xl mx-auto space-y-6`
> - Conteúdo offset: `margin-top: 70px; margin-left: 72px (mini) ou 260px (expandida)` no desktop
>
> **TABELAS**
> - `hidden sm:table-cell` (640px), `hidden md:table-cell` (768px), `hidden lg:table-cell` (1024px)
> - ID/nome e ações: sempre visíveis
> - Status: sempre visível
> - Container: `rounded-xl border border-border/50 bg-card overflow-hidden`

> - Tiles: `grid-cols-3 sm:grid-cols-4 md:grid-cols-5 lg:grid-cols-6`
> - Stats: `grid-cols-2 sm:grid-cols-4`
