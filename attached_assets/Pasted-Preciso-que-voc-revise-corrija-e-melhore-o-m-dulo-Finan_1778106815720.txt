Preciso que você revise, corrija e melhore o módulo Financeiro > Contas a Receber do meu sistema.

O objetivo desta solicitação é ajustar comportamento, filtros, paginação, UX/UI, impressão e coerência das abas para evitar informação errada, duplicada ou confusa para o colaborador responsável pelo financeiro.

IMPORTANTE:
- Não quebrar módulos existentes.
- Não remover funcionalidades sem necessidade.
- Não alterar regras de campanhas, vendedores, dashboard ou painel de TV.
- O módulo de Contas a Receber precisa ser extremamente claro, confiável, profissional e produtivo.
- O foco é evitar análise errada por filtro mal aplicado, popup repetido, layout inconsistente ou relatório mal formatado.

Mostrando 1–25 de 342 registros

====================================================
2. FILTROS INTELIGENTES
====================================================

Revisar todos os filtros do módulo de Contas a Receber para funcionar de forma inteligente e segura.

Regra principal:

Todo filtro em branco deve significar “Todos”.

Exemplos:

- Cliente vazio = todos os clientes
- Vendedor vazio = todos os vendedores
- Empresa vazia = todas as empresas
- Forma de pagamento vazia = todas as formas de pagamento
- Cidade vazia = todas as cidades
- UF vazia = todos os estados
- Status vazio = todos os status
- Valor mínimo vazio = sem valor mínimo
- Valor máximo vazio = sem valor máximo

Filtro de data:

O filtro principal do módulo deve sempre considerar DATA DE VENCIMENTO.

Se o período de vencimento estiver vazio:
- Considerar todo o período existente no banco/cache.
- Não limitar por mês atual automaticamente.
- Não esconder dados antigos ou futuros sem o usuário pedir.

Se apenas data inicial for preenchida:
- Filtrar vencimentos a partir da data inicial.

Se apenas data final for preenchida:
- Filtrar vencimentos até a data final.

Se data inicial e final forem preenchidas:
- Filtrar vencimentos dentro do intervalo.

A regra precisa ser clara no backend e no frontend.

Evitar filtros “fantasma”, ou seja:
- Filtro visual aparece limpo, mas ainda está aplicado internamente.
- Card clicado aplica filtro sem mostrar visualmente qual filtro está ativo.
- Troca de aba mantém filtro incompatível sem avisar.

Sempre mostrar filtros ativos em formato de chips/badges, por exemplo:
Filtros ativos:
- Status: Vencido
- Vendedor: João
- Vencimento: 01/05/2026 até 31/05/2026

Adicionar botão:
“Limpar filtros”

Esse botão deve realmente limpar todos os filtros e recarregar a visão padrão.

3. COMPORTAMENTO DOS CARDS E BOTÕES
====================================================

Percebi que ao clicar nos cards do topo ou em “Pendências” no módulo de Contas a Receber, o sistema sempre abre o mesmo popup.

Isso está desnecessário e confuso.

Revisar o comportamento dos cards e botões.

Regra desejada:

Cards do topo devem servir principalmente para FILTRAR a tela atual, não para abrir sempre o mesmo popup.

Exemplos:

Card “Total em Aberto”:
- Deve aplicar filtro de todos os títulos em aberto.

Card “Vencido”:
- Deve aplicar filtro status = VENCIDO.

Card “Vence Hoje”:
- Deve aplicar filtro vencimento = hoje.

Card “A Vencer”:
- Deve aplicar filtro status = A_VENCER.

Card “Clientes com Pendência”:
- Deve exibir clientes com saldo aberto.

Card “Maior Risco”:
- Pode abrir diretamente o detalhe do cliente de maior risco, se fizer sentido.

O botão “Pendências” deve ter função própria e clara.

Se já existir um botão específico para abrir detalhes ou popup, não repetir essa mesma ação nos cards.

Evitar duplicidade de função.

Cada elemento deve ter uma finalidade clara:

- Card = resumo + filtro rápido
- Botão Ver detalhes = abrir detalhe
- Botão Pendências = abrir pendências financeiras
- Botão Imprimir = gerar relatório
- Botão Exportar = exportar dados

4. COERÊNCIA ENTRE ABAS: CLIENTES, POR VENDEDOR E DUPLICATAS
====================================================

Revisar a lógica das abas do módulo:

- Clientes
- Por Vendedor
- Duplicatas
- Fila de Cobrança
- Aging, se existir

Tenho receio de informações erradas ou cruzadas entre filtros e abas.

Preciso que cada aba tenha comportamento lógico e coerente.

Aba Clientes:
- A visão principal deve ser agrupada por cliente.
- Filtros válidos:
  - Período de vencimento
  - Empresa
  - Cliente
  - Vendedor
  - Status
  - Cidade
  - UF
  - Forma de pagamento
  - Faixa de atraso
  - Valor mínimo/máximo

Aba Por Vendedor:
- A visão deve ser agrupada por vendedor.
- O filtro de vendedor pode existir, mas deve fazer sentido:
  - Se nenhum vendedor for selecionado, mostrar todos.
  - Se um vendedor for selecionado, mostrar somente ele.
- O filtro de cliente pode ser usado para localizar cliente dentro de vendedor.
- Não permitir cruzamento confuso onde parece estar analisando todos, mas está filtrado por cliente escondido.

Aba Duplicatas:
- Deve mostrar títulos individuais.
- Todos os filtros podem ser usados.
- É a aba analítica/detalhada.

Fila de Cobrança:
- Deve priorizar apenas títulos/clientes que precisam de ação.
- Por padrão, deve focar em vencidos, promessas vencidas, clientes críticos e sem cobrança.
- Se o usuário aplicar filtro, respeitar o filtro e deixar claro.

Aging:
- Deve sempre respeitar o período de vencimento selecionado.
- Se período estiver vazio, considerar todo o período existente.

Regras importantes:

Ao trocar de aba:
- Manter apenas filtros compatíveis.
- Se algum filtro não fizer sentido naquela aba, desativar visualmente ou ignorar com aviso claro.
- Nunca aplicar filtro escondido.
- Mostrar chips de filtros ativos.
- O usuário precisa entender exatamente o que está sendo analisado.

Se necessário, implementar filtros globais e filtros específicos por aba.
